# Issue #9 Monthly-review Evidence

Date: 2026-07-16
Status: approved

## Contract and grain

One review selection is one normalized calendar month. One
`monthly_review_action_summary_versions` row is one immutable, explicitly saved
HR conclusion for that month. The server owns month normalization, KPIs,
proposal derivation, version allocation, actor attribution, and workbook
content; React only selects a month and submits editable conclusion text.

## Authorization and concurrency

- Admin and editor can read, save, and export; viewer is forbidden.
- Saving requires CSRF and forbids client actor/version fields.
- The existing month-scoped advisory lock gives simultaneous successful saves
  distinct sequential versions. The latest saved actor and version are returned
  through the same read interface.
- Invalid months and forged or unauthorized requests use the stable error
  envelope without partial conclusions.

## Workbook and journey

- The export uses the selected month's canonical detail tables and the latest
  saved conclusion, falling back to the visibly separate server proposal only
  when no saved conclusion exists.
- Filename, OpenXML MIME type, private/no-store caching, eight sheet names,
  review-month title, and action-summary values are asserted on generated bytes.
- React exposes overview, complete detail tables, and summary/export modes.
  Saving refreshes actor/version state; download remains same-origin.
