# Issue #2 Read-Only React Slice Evidence

Date: 2026-07-15  
Status: implemented and verified

## Delivered boundary

- Same-origin React shell now renders a role-safe dashboard after server session
  revalidation.
- Admin/editor can open HR home, search and filter a server-paginated learner
  directory, and open learner current context, course history, and the approved
  learner audit summary without a document reload.
- Viewer receives the approved application summary, has no learner navigation,
  and receives `403 forbidden` from learner APIs. Viewer reports remain on the
  Streamlit surface until the reports tracer slice.
- React owns only these read surfaces. Existing Streamlit commands and
  `BusinessService` transaction boundaries are unchanged.

## Contract and data parity

- `api/learner_reads.py` owns parameterized filters, count, stable ordering, and
  pagination instead of loading the legacy 500-row snapshot into React.
- Fixed-fixture tests compare directory fields directly with
  `learner_directory_rows` and exercise detail/history/audit output.
- `scripts/check_issue2_read_parity.py` ran read-only against the representative
  workbook-loaded database: 365 legacy rows and 365 endpoint rows matched over
  employee identity, person/organization, current class/course/status,
  attendance, entrance level, and PIC fields.
- Audit response fields are limited to timestamp, actor username, and action;
  the legacy `details` JSON is intentionally not exposed.

## Automated evidence

- `python -m pytest tests/test_api_learner_reads.py -q`: 6 passed.
- `npm test`: 3 passed.
- `npm run build`: production Vite build passed.
- `npm run test:e2e`: 2 Playwright tests passed on Chrome. The admin test covers
  login, HR home, learner navigation, search, detail, course history, and audit
  summary. The viewer test proves summary visibility and absent HR navigation.
- OpenAPI is exported deterministically and TypeScript types are generated by
  `npm run api:generate`; `npm run api:check` detects committed contract drift.

## State coverage

- Loading: dashboard, directory, and detail announce scoped loading state.
- Empty: a successful empty page shows guidance to broaden or clear filters.
- Forbidden: route-level `403` plus absence of viewer HR navigation.
- Not found: authorized unknown employee returns stable `404 not_found`.
- Validation: invalid status/page size returns stable `422 invalid_input` with
  field errors.
- Unexpected failures: the existing common API handler returns a safe `500`
  envelope without exception or SQL detail.

## Manual browser observation

The local production build was checked through the connected Chrome browser at
desktop width. HR home, directory table, filtered single result, learner detail,
course history, and change history rendered correctly with no full-page
navigation. The browser session and temporary local server were closed after
verification.
