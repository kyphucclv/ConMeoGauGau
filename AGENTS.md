# Repository Agent Guidance

Follow `PROJECT_RULES.md` for every implementation task. Preserve the canonical
data model, transaction ownership, audit attribution, and no-silent-data-loss
rules.

## Agent skills

### Issue tracker

GitHub Issues; external pull requests are not a triage surface. See
`docs/agents/issue-tracker.md`.

### Triage labels

Human-led triage; `ready-for-agent` is disabled. See
`docs/agents/triage-labels.md`.

### Domain docs

Single-context layout using root `CONTEXT.md` and `docs/adr/`. See
`docs/agents/domain.md`.
